# 구현된 기능

## 전투 시스템
- 플레이어를 때리거나 공격당하면 30분 전투 상태 돌입
- 전투 트리거: 직접 타격, 플레이어가 쏜 모든 투사체(화살·삼지창·물약 등, 스켈레톤/스트레이 제외), TNT(플레이어 점화), 엔더수정·TNT 마인카트(DamageSource 귀속)
- 위 목록 이외의 피해(낙하, 화재, 익사 등)는 전투 트리거 아님
- 사망 시 양쪽 전투 해제
- 접속 종료 시 양쪽 전투 해제 + 해당 플레이어 즉시 사망
- `/pvpcancel` — 요청자가 입력하면 상대에게 [예]/[아니오] 확인 메세지 전송, 양쪽 수락 시 전투 취소
- `/pvpdeny` — 전투 취소 요청 거절

## 전투 UI
- **액션바**: 전투 중 매 초 "⚔ 전투 중" 표시. 명령어 잠금 중이면 남은 초도 함께 표시
- **보스바**: "⚔ 전투 중 | 나가면 죽습니다 | MM:SS" — 30분 카운트다운 포함
- **채팅**: 전투 돌입 시 명령어 안내 메세지 (본인에게만 전송)

## 명령어 제한
- **전투 아닐 때**: 모든 명령어 즉시 사용 가능
- **전투 중**: 마지막 전투 피해 후 15초가 지나야 명령어 사용 가능
  - 전투 시작 시점부터 타이머 시작
  - 전투 판정 피해를 받으면 15초 타이머 초기화 (낙하 등 일반 피해는 초기화 안 함)
  - 30분 경과 시 피해 여부와 관계없이 전투 해제
  - 명령어 차단 시 남은 초 표시

## TPA
- `/tpa <플레이어>` — 텔레포트 요청 (60초 만료)
- `/tpaccept` — 수락 후 요청자 즉시 텔레포트
- `/tpdeny` — 거절

## 집
- `/sethome <이름>` — 현재 위치 저장 (최대 3개)
- `/delhome <이름>` — 집 삭제
- `/home <이름>` — 집 목록 보기 또는 이동
- 집 데이터: `plugins/smp/homes.yml` 에 UUID별 저장, 재시작 후에도 유지

## 기타
- `/back` — 사망 후 1분 이내 마지막 죽은 위치로 이동
- `/logout` — 안전 퇴장. 전투 중이면 명령어 잠금 적용

## 채팅
- 전투 중 채팅 불가 (`[전투] 전투 중에는 채팅을 사용할 수 없습니다.` 메세지)
- 2초 채팅 쿨다운 (접속 후 첫 메세지는 면제)
- 직전 메세지와 동일한 내용 재전송 불가
- `config.yml`의 `blocked-messages` 목록에 있는 메세지 차단 (대소문자 무시)

## 언어 설정
- `config.yml`에서 `language: en` 또는 `language: kr`로 기본 언어 설정
- `/smplang <kr|en>` — 콘솔 또는 OP만 사용 가능, 즉시 적용 및 config.yml 자동 저장
